Azure Active Directory

Azure Active Directory holds all user acounts and grants default permissions. The permissions and roles depends on the user itself. This can be changed to allow or deny access to Azure resources. The same goes for Users and Groups also. Groups can be set up using default or custom permissions. Azure can also extend its users to that of guest users. Think of it as collaboration when you want to work with other users outside of your current organization. You can do this by simply inviting them.

Az Copy

You can also use Az copy to copy files or blobs to or from a storage account. This process can be done via command line and requires its own separate command line tool that can be installed and downloaded from online. 

At first I kept getting errors that it failed copying the file to the destination. So I used the SAS token method in order to successfully copy the file. Basically the command was the same but add the SAS token at the end of the file to copy like:

azcopy copy "C:\local\path" "https://account.blob.core.windows.net/mycontainer1/?sv=2018-03-28&ss=bjqt&srt=sco&sp=rwddgcup&se=2019-05-01T05:01:17Z&st=2019-04-30T21:01:17Z&spr=https&sig=MGCXiyEzbtttkr3ewJIh2AR8KrghSy1DGM9ovN734bQF4%3D" --recursive=true

The SAS token is highlighted in bold.

Template

Templates are a feature that allows users to create resources like virtual machines. Azure quickstart templates can do this. Deploy a custom template can do this as well. Simply search for template in portal search bar and either create a quickstart template or the other. You can also export a template from an existing virtual machine so you can redeploy later if neccessary. This is useful when you want to move your virtual machine to another virtual network. Once again the errors I encountered was due to the restrictions of my 'Free Trial' account and being limited to creating only certain vms.

Docker Instances

I was familiar with using Docker in the 'Virtualization' paper the previous semester so I was curious to see how it works with Azure. Like most resources it can be created using

- azure portal

- az cli

- powershell

The container can also be created using a 'container registry'. The container registry provides storage for docker images in the cloud similar to github and docker hub. You can download the source code for the container if you have it stored in github. From there you can build the image in the downloaded directory. But you must have an existing container registry. A web app can be created to retrieve the image from a repo in the container registry too. A webhook can be added to allow for continuous update on the web app whenver there is change. It will then be reflected on the web app.
