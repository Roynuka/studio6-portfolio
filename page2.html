Azure Active Directory
Azure Active Directory holds all user acounts and grants default permissions. The permissions and roles depends on the user itself. This can be changed to allow or deny access to Azure resources. The same goes for Users and Groups also. Groups can be set up using default or custom permissions. Azure can also extend its users to that of guest users. Think of it as collaboration when you want to work with other users outside of your current organization. You can do this by simply inviting them.

Storage Account

Azure has many storage options and Storage account is one of them. Alot of the labs I did requried a storage account and you need one for when you launch the 'Azure cloud shell' in the azure portal. The storage account have different settings like a lot of resources on Azure. It can be created using Az portal, cli, or powershell.

AZ CLI

Most of the preparation I did with the labs was done using AZ CLI. This was either on the portal and locally with the AZ CLI tool downloaded from the internet. I was more comfortable using azure commands on 'az cli' than powershell. 

AZ CLI - installed locally

The command line was downloaded and installed on my local machine. From there you launch the command line tool.

AZ CLI - Azure Portal

The cloud shell when launched on the portal. A storage account is required and will be prompted to create one if you're launching for the first time. The environment can be switched between 'bash' and 'powershell'

AZ CLI - Login

The 'az login' command is used to connect to azure using your credentials. From there you're taken to the login screen. Once authenticated you will be able to run az commands on your command line

Storage accounts provide a way to store data, files, tables, and messages. Azure storage explorer makes it easier to manipulate the data. It allows to manage multiple storage accounts, use the interface similar to file explorer, and is free to download and use.

Connect to Azure Storage

After downlaoding storage explorer and running. You can connect to your account on azure portal

Login

Login using credentials

Az Copy

You can also use Az copy to copy files or blobs to or from a storage account. This process can be done via command line and requires its own separate command line tool that can be installed and downloaded from online. 

AZ copy

Once downloaded and extracted open 'terminal' in the directory and login using 'az login' and follow prompts

Log in

If logging in for the first time you can use 'az login --tenant-id <tenant id>. You will be prompted to follow a link and verify login with a code on browser. Afterwards you can be able to login.

Az copy to container

I used Az copy to copy a txt file to a container. Using the terminal for az copy you need the location of the destination, in this case the blob container. This can be found under the properties tag of the container
